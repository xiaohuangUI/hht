# 第二轮精修：模块拆解与推进节奏

## 一、拆解原则
- 先稳定主线：先做“能展示、能解释、能复现”的核心链路。
- 分段交付：每个模块都要有独立可验收结果，不互相等待。
- 风险前置：高风险项提前验证，避免临近提交时集中爆雷。

## 二、模块拆解（M1-M8）
1. M1 数据底座与治理  
   目标：导入、去重、台账、导出可复现。
2. M2 基线模型实验室  
   目标：随机森林/梯度提升/感知机对比与解释。
3. M3 深度预测与异常检测  
   目标：多步预测、异常识别、可解释图表。
4. M4 策略推演与问答联动  
   目标：参数调节 -> 风险变化 -> 策略建议闭环。
5. M5 应急站选址优化  
   目标：复杂网络建模、选址优化、覆盖评估。
6. M6 本地训练闭环  
   目标：数据检查、训练命令、训练记录回看。
7. M7 视觉检测专项（当前暂缓）  
   目标：暂时保留接口与数据准备，不作为当前主线。
8. M8 答辩演示与交付  
   目标：演示脚本、录屏流程、最终提交包。

## 三、一周推进建议
1. 第1-2天：M1 + M2（清数据、跑基线、固化指标）
2. 第3-4天：M3 + M4（深度预测、策略联动、问答完善）
3. 第5天：M5（应急站选址图表和结论打磨）
4. 第6天：M6（本地训练闭环验证，GPU/CPU流程跑通）
5. 第7天：M8（答辩脚本彩排、成果打包）

## 四、验收标准（最小可交付）
1. 每个模块至少有1个可视化结果 + 1段文字解读。
2. 所有关键模块都可在本地复现实验。
3. 训练与推演流程都有日志或记录可回溯。
4. 页面无明显报错，关键路径可在10分钟内演示完整。
